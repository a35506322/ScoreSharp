# 原持卡人定義

## 欄位定義

- 欄位名稱：IsOriginalCardholder
- 資料類型：字串
- 資料長度：1
- 資料說明：
- **Y**: 原持卡人
- **N**: 非原持卡人

## 判斷時間點

### 2. 待原始卡人查詢\_快發 A02（批次作業）

針對快發 A02 案件執行的批次查詢作業。

### 3. 線上案件於月收入確認完成 (TODO: 尚未實作)

當線上申請案件完成月收入確認後，系統會進行原持卡人判斷。

### 4. 紙本案件於月收入確認完成 (TODO: 尚未實作)

紙本申請案件完成月收入確認後，案件狀態會更新為「初始」，接續執行步驟 1 的批次作業。

## 判斷邏輯

### 條件

利用 MW3 查詢原持卡人，並判斷是否為原持卡人。

- 存在客戶 ID（有 ID 記錄）
- **且** 信用額度 >= 5,000 元（信用額度為 0 則視為非原持卡人）

### 結果

- **符合上述條件** → 【原持卡人】= Y（原持卡人）
- **不符合上述條件** → 【原持卡人】= N（非原持卡人）
