# 重複進件定義

## 欄位定義

- 欄位名稱：IsDuplicateApplication
- 資料類型：字串
- 資料長度：1
- 資料說明：
  - **Y**: 是（重複進件）
  - **N**: 否（非重複進件）

## 判斷時間點

### 1. 網路件原卡友檢核（批次作業）

### 2. 紙本件檢核（批次作業）

### 3. 網路件非卡友檢核（批次作業）

### 4. 線上案件於月收入確認完成 (TODO: 尚未實作)

當線上申請案件完成月收入確認後，系統會進行重複進件判斷。

### 5. 紙本案件於月收入確認完成 (TODO: 尚未實作)

紙本申請案件完成月收入確認後，案件狀態會更新為「初始」，接續執行步驟 1 的批次作業。

## 判斷邏輯

### 條件

利用 ID 查詢資料庫，並判斷是否為重複進件。

**必須同時符合以下一個條件：**

**重複申請檢查**：

- 已有同一客戶 ID 的其他申請編號存在於資料庫中

### 結果

- **符合上述所有條件** → 【重複進件】= Y（是）
- **不符合上述條件** → 【重複進件】= N（否）

## 適用範圍

- 紙本進件
- 網路進件
